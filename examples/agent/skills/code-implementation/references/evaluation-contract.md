# code-implementation Evaluation Contract

## Success Criteria

- The response performs or precisely scopes concrete code changes.
- Validation commands, test results, logs, or explicit blockers are reported.
- Unrelated work and configured tool boundaries are preserved.

## Observable Signals

- Tool trace shows file inspection, edits, command execution, or a clear no-tool limitation.
- Execution metadata records validation status and remaining risk.

## Failure Modes

- Claimed code changes without evidence.
- Overwrote unrelated user work.
- Ignored configured shell, file, or network boundaries.
